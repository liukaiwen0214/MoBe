# 项目技术说明文档

## 1. 项目概述

### 1.1 项目简介
MoBe (Life OS) 是一个综合性的个人生活管理系统，旨在帮助用户管理个人生活的各个方面，包括财务管理、日程安排、任务管理、习惯养成等。

### 1.2 项目结构
项目采用前后端分离架构，分为前端和后端两个主要部分：

- **前端**：`life-os-web` 目录，基于 Nuxt.js 4 框架开发
- **后端**：`life-os-server` 目录，基于 Spring Boot 4 框架开发
- **数据库**：MySQL 数据库，SQL 初始化脚本位于 `sql` 目录

### 3.2 核心流程

1. **用户认证流程**：
   - 用户通过前端登录页面提交登录请求
   - 后端验证用户凭据并生成 JWT 令牌
   - 前端存储令牌并用于后续请求的认证

2. **数据请求流程**：
   - 前端通过 Axios 发送请求到后端 API
   - 请求拦截器添加认证令牌
   - 后端验证令牌并处理请求
   - 后端返回响应数据
   - 前端响应拦截器处理响应

## 4. 前端技术实现

### 4.1 项目结构

```
life-os-web/
├── app/
│   ├── api/              # API 接口定义
│   ├── assets/           # 静态资源
│   ├── components/       # 组件
│   ├── composables/      # 组合式函数
│   ├── layouts/          # 布局组件
│   ├── middleware/       # 中间件
│   ├── pages/            # 页面
│   ├── stores/           # 状态管理
│   ├── utils/            # 工具函数
│   ├── app.config.ts     # 应用配置
│   └── app.vue           # 根组件
├── public/               # 公共资源
├── nuxt.config.ts        # Nuxt 配置
├── package.json          # 项目依赖
└── tsconfig.json         # TypeScript 配置
```

### 4.2 路由管理

前端使用 Nuxt.js 的文件系统路由，页面文件直接对应路由路径：

- `/pages/login.vue` → `/login`
- `/pages/profile.vue` → `/profile`
- `/pages/settings/password.vue` → `/settings/password`

### 4.3 状态管理

使用 Pinia 进行状态管理，核心状态包括：

- **用户状态**：管理用户认证信息、用户详情等
- **应用状态**：管理应用配置、主题等

### 4.4 请求封装

前端使用 Axios 封装 HTTP 请求，主要功能包括：

- 基础 URL 配置
- 请求超时设置
- 请求拦截器：添加认证令牌
- 响应拦截器：统一处理响应数据和错误

### 4.5 页面组件设计

主要页面组件包括：

- **认证相关**：登录页、注册页
- **用户相关**：个人资料页、设置页
- **功能模块**：财务管理、日历、任务管理、习惯追踪、笔记等

### 4.6 中间件

- **auth**：验证用户是否已登录，未登录则重定向到登录页
- **guest**：验证用户是否未登录，已登录则重定向到首页

### 5.2 接口设计规范

- **URL 格式**：`/api/v1/{module}/{action}`
- **HTTP 方法**：GET、POST、PUT、DELETE
- **响应格式**：统一使用 `ApiResponse` 包装
- **参数验证**：使用 Spring Validation 进行请求参数验证

### 5.3 业务逻辑实现

核心业务模块：

- **用户管理**：用户注册、登录、个人信息管理
- **财务管理**：财务记录的增删改查
- **会话管理**：用户会话的创建和管理

### 5.4 核心方法与属性

- **用户认证**：`UserService.login()`、`UserService.register()`
- **令牌生成**：`TokenUtil.generateToken()`
- **密码加密**：`PasswordConfig.passwordEncoder()`
- **权限验证**：`AuthUtil.verifyToken()`

### 5.5 异常处理

- **全局异常处理器**：`GlobalExceptionHandler`
- **业务异常**：`BizException`

### 5.6 权限控制

- **JWT 令牌验证**：通过请求头中的 `Authorization` 字段验证
- **用户拥有权验证**：确保用户只能访问自己的数据

### 6.2 索引设计

- **用户表**：
  - PRIMARY KEY (`id`)
  - UNIQUE KEY `uk_user_username` (`username`)
  - UNIQUE KEY `uk_user_email` (`email`)
  - KEY `idx_user_status` (`status`)
  - KEY `idx_user_deleted` (`is_deleted`)

- **验证码表**：
  - PRIMARY KEY (`id`)
  - KEY `idx_auth_codes_target` (`target`)
  - KEY `idx_auth_codes_type` (`type`)
  - KEY `idx_auth_codes_expires_at` (`expires_at`)
  - KEY `idx_auth_codes_deleted` (`is_deleted`)

- **财务记录表**：
  - PRIMARY KEY (`id`)
  - KEY `idx_finance_records_user_id` (`user_id`)
  - KEY `idx_finance_records_type` (`type`)
  - KEY `idx_finance_records_category` (`category`)
  - KEY `idx_finance_records_record_date` (`record_date`)
  - KEY `idx_finance_records_deleted` (`is_deleted`)

- **用户偏好设置表**：
  - PRIMARY KEY (`id`)
  - UNIQUE KEY `uk_user_preferences_user_id` (`user_id`)
  - KEY `idx_user_preferences_deleted` (`is_deleted`)

- **用户会话表**：
  - PRIMARY KEY (`id`)
  - UNIQUE KEY `uk_user_session_token` (`session_token`)
  - KEY `idx_user_session_user_id` (`user_id`)
  - KEY `idx_user_session_expires_at` (`expires_at`)
  - KEY `idx_user_session_status` (`status`)
  - KEY `idx_user_session_deleted` (`is_deleted`)

### 6.3 关联关系

- **user** 与 **user_preferences**：一对一关系
- **user** 与 **user_session**：一对多关系
- **user** 与 **finance_record**：一对多关系

### 6.4 SQL 规范

- 使用 `utf8mb4` 字符集
- 使用 `InnoDB` 存储引擎
- 遵循命名规范：表名使用小写，单词间用下划线分隔
- 字段名使用小写，单词间用下划线分隔
- 合理使用索引，避免过度索引

## 8. 部署方式

1. 构建生产版本：`npm run build`
2. 部署构建产物到静态文件服务器或 CDN

1. 构建 JAR 包：`mvn clean package`
2. 部署到 Java 应用服务器（如 Tomcat）或使用 `java -jar` 直接运行

## 9. 接口文档

### 9.1 认证接口

- **获取验证码**：`GET /api/v1/auth/captcha`
- **登录**：`POST /api/v1/auth/login`
- **发送注册验证码**：`POST /api/v1/auth/register/send-code`
- **注册**：`POST /api/v1/auth/register`

### 9.2 用户接口

- **获取当前用户信息**：`GET /api/v1/user/me`
- **更新用户资料**：`PUT /api/v1/user/profile`
- **更新密码**：`PUT /api/v1/user/password`
- **更新用户偏好**：`PUT /api/v1/user/preferences`

### 9.3 财务管理接口

- **创建财务记录**：`POST /api/v1/finance/record`
- **获取财务记录列表**：`GET /api/v1/finance/record`
- **更新财务记录**：`PUT /api/v1/finance/record/{id}`
- **删除财务记录**：`DELETE /api/v1/finance/record/{id}`
